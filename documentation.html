💡 Konsep

┌─────────────────┐ │ User Service │ ← Satu-satunya tempat login │ :8000 │ & database user └────────┬────────┘ │ │ Token diverifikasi ke sini │ ┌────────┴──────────────────────────┐ │ │ │ ┌──────────┐ ┌──────────┐ │ │ │ HRM │ │ Ticket │ │ ← Service lain TIDAK │ │ :8001 │ │ :8002 │ │ perlu database user │ └──────────┘ └──────────┘ │ │ │ │ User login SEKALI, │ │ bisa akses SEMUA service │ └────────────────────────────────────┘
✅ Keuntungan:
  • Single Sign-On (SSO) - login sekali akses semua
  • Tidak perlu database user di setiap service
  • Tidak perlu sharing JWT secret key
  • User management terpusat
  • Logout sekali langsung berpengaruh ke semua service

🚀 Installasi

1. Install di Aplikasi Anda

# Di project HRM
cd /var/www/hrm
composer require bpjs/auth-client

# Di project Ticket
cd /var/www/ticket
composer require bpjs/auth-client

⚡ Quick Start

1. Setup .env

# .env di setiap aplikasi
AUTH_SERVICE_URL=http://192.168.1.10:8000
AUTH_APP_KEY=your_app_key
AUTH_APP_SECRET=your_app_secret

2. Tambahkan Middleware

<?php
// routes/api.php

use Bpjs\AuthServiceClient\Middleware\AuthMiddlewareClient;

// ✅ HANYA 3 BARIS!
Api::group([AuthMiddlewareClient::class], function() {
    Api::get('/payroll', [PayrollController::class, 'index']);
    Api::post('/transaction', [TransactionController::class, 'store']);
});

3. Pakai User Data di Controller

<?php

class PayrollController
{
    public function index()
    {
        // ✅ User sudah ada di session
        $user = $_SESSION['user'];
        
        echo "Welcome, " . $user['username'];
        // Output: Welcome, admin
    }
}
🎉 Selesai! Tidak perlu kode autentikasi lagi!

🔄 Cara Kerja

Flow Detail

1 USER LOGIN (sekali)
Browser ──POST──► User Service :8000/api/auth/login │ ▼ Generate JWT Token │ ▼ Browser ◄──{ token: "eyJ...", user: {...} }── User Service
2 AKSES HRM
Browser ──GET──► HRM Service :8001/api/payroll Header: Authorization: Bearer eyJ...
3 MIDDLEWARE VERIFIKASI
AuthMiddlewareClient::handle() │ ▼ Ambil token dari header │ ▼ HTTP POST ke User Service ► /api/auth/verify-token │ ▼ User Service cek token │ ▼ Return user data {id, username, roles, permissions} │ ▼ Simpan di $_SESSION['user'] │ ▼ Lanjut ke Controller
💡 Dengan Cache: Request kedua dengan token yang sama akan langsung ambil dari cache, tanpa HTTP request ke User Service (lebih cepat 200x!)

📘 Penggunaan

A. Sebagai Middleware (Cara Paling Simpel)

<?php
// routes/api.php

use Bpjs\AuthServiceClient\Middleware\AuthMiddlewareClient;

// Route public (tanpa middleware)
Api::post('/auth/login', [AuthController::class, 'login']);

// Route protected (dengan middleware)
Api::group([AuthMiddlewareClient::class], function() {
    
    Api::get('/dashboard', [DashboardController::class, 'index']);
    Api::get('/payroll', [PayrollController::class, 'index']);
    Api::post('/transaction', [TransactionController::class, 'store']);
    
});

B. User Data di Controller

<?php

class PayrollController
{
    public function index()
    {
        // User dari session
        $user = $_SESSION['user'];
        $userId = $_SESSION['user_id'];
        $username = $_SESSION['username'];
        $permissions = $_SESSION['permissions'];
        
        // Cek permission
        if (!in_array('payroll_view', $permissions)) {
            return Response::json(['error' => 'Forbidden'], 403);
        }
        
        // Bisnis logic
        $data = Payroll::where('user_id', $userId)->get();
        
        return Response::json(['data' => $data]);
    }
}

C. Cek Permission & Menu

<?php

use Bpjs\AuthServiceClient\AuthServiceClient;

$auth = new AuthServiceClient([...]);

// Cek menu akses
if ($auth->checkMenuAccess($userId, 'payroll')) {
    echo "<a href='/payroll'>Payroll</a>";
}

// Cek permission
if ($auth->hasPermission($userId, 'employee_edit')) {
    echo "<button>Edit</button>";
}

// Ambil semua menu
$menus = $auth->getUserMenus($userId, 'hrm');
foreach ($menus as $menu) {
    echo $menu['name'];
}

D. Frontend (JavaScript)

// Login
async function login(username, password) {
    const res = await fetch('http://192.168.1.10:8000/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password })
    });
    
    const data = await res.json();
    localStorage.setItem('token', data.data.token);
    return data;
}

// Akses API dengan token
async function fetchPayroll() {
    const token = localStorage.getItem('token');
    
    const res = await fetch('http://192.168.1.10:8001/api/payroll', {
        headers: {
            'Authorization': `Bearer ${token}`
        }
    });
    
    if (res.status === 401) {
        window.location.href = '/login';
    }
    
    return res.json();
}

🐛 Error Handling

Response Format

Sukses (200)

{
    "status": "success",
    "data": {
        "id": 1,
        "username": "admin",
        "roles": ["admin"],
        "permissions": ["payroll_view"]
    }
}

Token Tidak Ditemukan (401)

{
    "status": 401,
    "message": "Token tidak ditemukan",
    "error": "unauthorized"
}

Token Expired (401)

{
    "status": 401,
    "message": "Token expired",
    "error": "token_expired",
    "code": "token_expired"
}

Forbidden (403)

{
    "status": 403,
    "message": "Anda tidak memiliki akses",
    "error": "forbidden"
}

🔧 Troubleshooting

⚠️ Token tidak ditemukan

Penyebab: Token tidak dikirim atau format salah

Solusi: Pastikan header Authorization dikirim dengan format Bearer <token>

❌ cURL Error: Failed to connect

Penyebab: User Service tidak bisa dihubungi

Solusi:

  1. Cek User Service running: curl http://192.168.1.10:8000/health
  2. Cek firewall
  3. Cek env AUTH_SERVICE_URL
⚠️ Invalid token

Penyebab: Token salah atau sudah expired

Solusi: Login ulang untuk mendapatkan token baru
